# 游戏开发项目

这是一个 HTML 小游戏开发项目，开发者是10岁的男孩。

## 沟通规则

- **所有回复必须用中文**
- 专业名词（technical terms）要用括号加英语说明，例如：变量（variable）、函数（function）
- 解释要简单易懂，像和小朋友说话一样
- 遇到复杂概念，用生活中的例子来说明
- 鼓励学习，保持积极正面的语气

## 用户偏好
- 用户对UI/游戏画面要求比较高，像素风格，分辨率需要高点
- 游戏角色需要有动画，不要一动不动
- 哥特童话风

## 项目类型

- 纯前端（frontend）网页游戏
- 使用 HTML、CSS、JavaScript 三件套
- 不依赖后端服务器（server）或数据库（database）
- 游戏文件可以直接在浏览器（browser）中打开运行

## 开发规范

### 文件结构（File Structure）
```
game1/
├── index.html      # 游戏主页面
├── style.css       # 样式表，控制游戏外观
├── game.js         # 游戏主逻辑代码
└── assets/         # 素材文件夹（图片、音效等）
```

### 代码风格（Code Style）
- 变量（variable）和函数（function）名用中文拼音或简单英文命名
- 每段代码都要加中文注释（comment）解释用途
- 保持代码简单，优先让孩子能看懂
- 不用太复杂的写法，一步一步来

### 游戏开发优先级
1. 先让游戏能运行起来（功能优先）
2. 再让游戏好玩（趣味性）
3. 最后让游戏好看（界面美化）

## 技术要求

- **不使用**框架（framework）如 React、Vue 等，保持纯原生（vanilla）代码
- **不使用** TypeScript，只用普通 JavaScript
- 优先使用 Canvas（画布）或 DOM 操作来制作游戏
- 游戏逻辑要写得清晰，方便孩子学习

## 解释代码时的规则

- 每次写新代码，都要解释这段代码做了什么
- 用"就像..."的方式打比方
- 鼓励孩子提问，没有蠢问题

## 常见游戏类型参考

- 打砖块（Breakout）
- 贪吃蛇（Snake）
- 迷宫（Maze）
- 接球游戏（Catch）
- 跑酷（Runner）
- 问答游戏（Quiz）

## 注意事项

- 游戏内容要适合10岁儿童，不包含暴力或不良内容
- 操作要简单，鼠标点击或键盘方向键控制
- 游戏要有明确的开始、进行中、结束三个状态
- 加入分数（score）系统让游戏更有趣
