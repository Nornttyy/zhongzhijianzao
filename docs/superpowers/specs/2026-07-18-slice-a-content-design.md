## 2. sim 层扩展（纯逻辑，可测）

### 2.1 状态结构

```ts
SimState { time, player, world }
WorldState {
  nodes: ResourceNode[]        // 树与矿，见 §2.2
  posts: Vec2[]                // 已放置提灯柱
  phantom: PhantomState        // §2.4
  inventory: { wood, fluorite }
  serenity: number             // 0..100，初始 100
  lost: boolean                // 迷失表现层标记
  placing: boolean             // 放置模式
  seed: number                 // 幻影随机数种子（mulberry32，状态内串行）
}
ResourceNode { id, kind: 'tree'|'ore', pos, charges }  // 树 4 次、矿 5 次，0 为耗尽
```

所有接口字段 `readonly`（收编遗留项）。篝火位置为常量入 config（唯一、不入状态）。

### 2.2 场景布局（config 固定坐标，非随机）

40×40 米。营地在正中：篝火 (20, 19)，玩家出生 (20, 20.8)。树 6 棵分布在离营地 7–12m 环带、矿 3 处 10–14m（更深更黑，用微光引导）：

- 树：(12.5, 13) (27, 11.5) (31.5, 22) (9, 25.5) (15.5, 31) (25.5, 29.5)
- 矿：(7.5, 16.5) (33, 15.5) (21.5, 34.5)

固定布局理由：确定性可测、可手工排布避免重叠、首个切片不需要随机地图。

### 2.3 采集判定与事件

保留现有"点左键即挥砍"的响应手感（可空挥）。**收益判定在 sim 侧**：某 tick 内 `gatherT` 跨越 `hitAt` 时，取交互半径 `gather.rangeM=1.6` 内最近的未耗尽节点；命中则 `charges-1`、背包 +1，发 `harvest` 事件（含 kind/nodeId/pos/depleted）。无目标则无事件（空挥，动画层照常敲击）。

sim 事件机制：`stepWorld` 返回 `[state, events]`；`Sim.advance` 聚合各步事件，`drainEvents()` 供 main 每帧消费。渲染层动画器的 `gatherHit`（视觉拍点）与 sim `harvest`（收益结算）分流：前者管敲击火花与敲击声，后者管萤火虫/晶屑、入包音与计数跳动。

### 2.4 幻影状态机

单实体，字段 `{ pos, mode: 'wander'|'stare'|'fade'|'gone', modeT, alpha, target }`。

- **wander**：0.6 m/s 走向随机路标（营地 10–18m 环带内取点，出界重取）；到达即取下一个。
- **stare**：玩家进入 8m（`stareRange`）→ 停下注视。安宁值额外 -2/s，压迫低鸣增益随距离升高（渲染层读距离）。玩家退出 9m（滞回）→ 回 wander。
- **fade**：玩家进入 6m（`dissolveRange`）→ 1.2s 内 alpha→0，进入瞬间发一次 `phantomSigh` 事件（轻叹）。
- **gone**：淡出完成后隐身 6s，再于"距玩家 ≥12m 且在环带内"的随机点重生，1s 淡入回 wander。

随机数用 `seed` 串行的 mulberry32，测试可复现。

### 2.5 安宁值结算

每 tick 取玩家所处最高档位（提灯柱/篝火光圈 +5/s ＞ 随身提灯 -0.5/s ＞ 黑暗 -3/s；本切片玩家恒带提灯，黑暗档为完备性保留并测试），幻影 stare 且玩家在 8m 内时叠加 -2/s。0..100 夹紧。

迷失滞回：<30 置 `lost` 并发 `lostEnter`；≥40 清除并发 `lostExit`。**纯表现层**，不影响操作；安宁值到 0 无惩罚（本作无失败态）。

### 2.6 合成与放置（E 键实装）

- `IntentInput` 增 `craft: boolean` 边沿；Keyboard 对 KeyE 锁存（与 interact 同款：缓存到实际步进帧、blur 清除）。
- 靠近篝火 `craftRange=2.5m` 且 木≥10 萤≥5：E → 扣资源、`placing=true`、发 `crafted`。
- placing 中：预览位 = 玩家前方 `placeAheadM=1.5m`（`x + facing*1.5`，夹紧世界边界内 1m）；E → 落地入 `posts`、发 `postPlaced`（含序号，首座触发达成提示）、`placing=false`。放置不做碰撞校验（YAGNI）。
- placing 中靠近篝火不再显示合成提示（放置提示优先）。

### 2.7 步进顺序

玩家步进 → 采集收益 → E（放置优先于合成）→ 幻影 → 安宁值/迷失。单处 `stepWorld` 编排，各子步骤纯函数。

## 3. 渲染层扩展

### 3.1 灯光世界坐标化（收编遗留项）

`LightSpec` 改为世界米坐标 + `alpha`（0..1 光洞强度），LightLayer 内部经 world 容器变换转屏幕像素。光源清单：随身提灯 3.5m（跟插值玩家位）、篝火 6m、每座提灯柱 5m、**萤石矿微光 1.2m/alpha 0.35**、**低语木微光 0.9m/alpha 0.25**（GDD"发光的低语木/提供光源的萤石"，兼作黑暗中的寻路引导；纯装饰，不参与安宁值档位）。耗尽节点微光熄灭。

### 3.2 世界实体渲染（新 `worldView.ts`)

树/矿/篝火/提灯柱/幻影建 Sprite，沿用"脚底锚点 + zIndex=yPx"排序。目标高度：树 3.2m、矿 1.1m、篝火 1.3m、提灯柱 2.2m、幻影 1.8m。全部素材缺失时程序占位（沿用 textures.ts 模式，六图全兜底）。

- **耗尽表现**（无树桩/暗矿素材，程序化降级）：树 → 压扁至 35% 高度并压暗（残桩剪影）；矿 → 压暗去饱和（tint 灰蓝）。素材后续到位再替换。
- **受击反馈**：harvest 事件触发目标短促摇晃（0.3s 衰减旋转脉冲，渲染层状态）。
- **幻影**：additive 混合（黑底素材），置于**暗幕之上**的独立屏幕层（自发光体不受暗幕遮蔽，远处黑暗中可见微光人形，符合 GDD"远处闪烁的灰白雾气人形"），alpha 来自 sim。
- **篝火**：木堆素材 + 程序火焰（additive 径向光斑呼吸 + 偶发火星粒子），"未点燃素材 + 程序点燃"。

### 3.3 迷失表现

屏幕边缘雾圈（canvas 径向渐变 vignette，屏幕层）+ 世界容器 ColorMatrixFilter 降饱和。渲染侧强度值以 ~1.5/s 速率向目标（lost?1:0）渐变，进出平滑。UI 层不受滤镜影响。

### 3.4 粒子扩展

- 树收益：金色萤火虫（上飘、缓慢摆动、1.5s 长命）。
- 矿收益：蓝白晶屑（短促四散）。
- 篝火：偶发上升火星。

## 4. 音频扩展（全程序合成，无音频文件）

- **主链路**：全部节点过 master 低通（常态 18kHz）；迷失时 0.8s 滑至 700Hz，解除滑回——"音频闷化"。
- **风**：循环噪声缓冲过低通 ~400Hz，慢速 LFO 起伏，低音量常驻。
- **低鸣**：~55Hz 三角波对（轻微失谐），增益由 main 每帧传入的"注视强度"（0..1，距离映射）平滑跟随。
- **轻叹**：phantomSigh 事件触发，带通噪声 500→300Hz 滑落 0.7s 包络。
- **收益音**：树=木质轻响+高频微铃；矿=双正弦晶莹叮。合成成功=风铃琶音；放置=暖冲击+铃。
- **重 arm**（收编遗留项）：visibilitychange/pointerdown 时若 context suspended 则 resume。

## 5. UI 层（新 `ui.ts`，屏幕层，位于暗幕之上）

- **背包计数**（右下）：木/萤石图标（程序 Graphics）+ 数字，入包时跳动。
- **安宁值**（左下）：程序化蒲公英——花头 + 12 根绒毛辐条，绒毛数 = ceil(serenity/100×12)，低值时逐根消失、色调转灰白（GDD 蒲公英 UI 的极简版）。
- **情境提示**（下中）：`deriveHint(state)` 纯函数产出当前提示文案（"左键 采集低语木"/"E 合成 提灯柱（木10 萤5）"/"E 放置提灯柱"/无），暗底圆角条。
- **提示浮层**（上中）：toast 队列（淡入-停留-淡出）。开场两条（"夜很深，跟随微光。""WASD 移动"），首座提灯柱达成提示（"第一盏灯亮起，森林安静了些。"），后续放置简报。
- 中文文案，系统字体。
